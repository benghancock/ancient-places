Overview & Motivation

The aim of this project is to allow archaeology enthusiasts, curious travelers, and amateur historians the ability to search through the Pleiades database using modern-day country names. For the unfamiliar, Pleiades is a public repository or “gazetteer” of geographic information about the ancient world. It’s an amazing project, and it offers very robust search already. But country-based search is one missing feature, and is enabled in this project by combining data from Pleiades with data from Natural Earth.

This project was also a way for me to get familiar with new-to-me technologies using real-world data, particularly the Echo web framework for Golang, and the PostGIS extension to the PostgreSQL database. All of which is to say, while I hope that the end product is useful to those who may chance upon this humble project, this was also undertaken for fun and learning.

One important note is that the Pleiades places returned when searching by country name do not necessarily fall exclusively within that country’s borders; only a part of the place may overlap with that country’s territory. Features spanning a large geographic area, (e.g. the Ganges River), will appear in searches for a variety of countries. In other words, places have a one-to-many relationship with modern day countries. Please also note that results may not always be correct; see the caveats mentioned below.

If you are viewing this document online, you can view a guide for how the data for this project was prepared on the guide page. Or view the documentation in the code repository (see below).

License & Caveats

I made efforts to check that my methods in combining the two data sets produced expected results. That said, I cannot vouch for the accuracy or completeness of the data – including whether the listed places actually fall within the named countries. When inspecting the data, I found instances where places fell into no country at all (usually when located next to a body of water) or into an unexpected country. For more notes on this, see the guide.
